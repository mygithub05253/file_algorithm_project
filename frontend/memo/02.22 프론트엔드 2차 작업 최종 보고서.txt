2차 기능 구현 보고서


1) 수정 및 추가 내용 요약
1. 카테고리창에서 사용자가 추가 및 삭제를 하여 관리 할수있게 수정함.
2. 로그아웃기능을 추가하여 표지로 이동할 수 있게 추가함.
3. 파일에 각각 날짜를 표기하여 파일명이 겹치지 않게 수정함.
4. 전체선택, 전체해제, 전체 다운로드, 선택다운로드, 선택삭제 버튼의 기능이 작동하도록 수정함.
5. 최신순, 오래된순 정렬기능을 추가하여 작동되도록 수정함.


2) System.html 보안된 부분
1. 로그아웃 버튼 추가
- id="logoutButton"
2. 카테고리 기능 추가
- 새 카테고리명을 입력할 수 있는 input 태그 (id="newCategoryName") 추가됨.
- createCategory() 버튼이 추가되어 새로운 카테고리를 만들 수 있도록 개선됨.


3) system.css
1. #logoutButton의 디자인 추가


4)system.js
1. 카테고리 추가 및 삭제 기능
- createCategory() 함수를 통해 사용자가 원하는 폴더를 직접 추가 가능.
- removeCategory() 함수를 통해 불필요한 폴더를 삭제 가능.

2. 로그아웃 기능
- logout() 함수를 통해 로그아웃 버튼이 눌리면 표지로 이동하게 함.

3. 파일에 날짜 표기 기능
- displayFiles() 함수에서 업로드 날짜를 표시하여 파일 정렬 및 관리가 쉽게 함.
- getSelectedFiles() 함수로 체크된 파일들을 가져와 한 번에 다운로드/삭제 가능.

4.전체선택, 전체해제, 전체 다운로드, 선택다운로드, 선택삭제
- selectAllFiles()에서 전체 파일 선택 가능
- deleteSelected()에서 체크된 파일을 한 번에 삭제 가능.
- downloadSelected()에서 체크된 파일을 한 번에 다운로드할 수 있음.
- downloadAll()을 통해 현재 탭의 모든 파일을 한 번에 다운로드 가능.

5. 최신순, 오래된순 정렬기능 구현
- sortFiles()에서 최신순과 오래된순을 계산해서 정렬되게 함.

2차 프론트엔드 피드백

1. 불필요한 버튼과 기능이 너무 많습니다. 추후 개발하면서 제거 부탁드립니다.
2. 파일을 등록 후 파일들을 폴더 형식으로 분류하고 나눠야 합니다.
이것을 위해서 등록한 파일들이 뜨는 목록에서 파일 간 다른 폴더로 분류되는 기능을 추가해주세요.
단 가시성이 좋아야하니 좀 명확한 구분이 가능하게 부탁드립니다.
