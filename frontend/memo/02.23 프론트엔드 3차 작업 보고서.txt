1. 추가 및 변경 요구 사항
1) 파일을 업로드할 때 특정 폴더에 자동으로 분류되지 않는 문제 발생
2) 파일명 기반 자동 분류 알고리즘 개선 및 기존 기능 유지하면서 보다 직관적인 폴더 관리 가능 요구

2. 기능 개선 사항
1) "기타 파일" 폴더 추가 및 자동 분류 개선
- "기타 파일" 폴더가 기본적으로 제공되며 삭제 불가능
- 기존 "전체 파일" 폴더와 함께 기본 폴더로 유지
- 파일명이 기존 폴더명과 일치하는 것이 없는 경우, "기타 파일"과 "전체 파일"에 자동 추가

2) 파일 자동 분류 알고리즘 로직 수정
- 파일명이 폴더명과 완전히 일치하는 경우에만 해당 폴더로 이동했었음
- 파일명이 폴더명을 포함할 경우에도 자동으로 해당 폴더에 추가되도록 수정

3) 폴더(카테고리) 추가 및 관리 기능 복구/수정
- 사용자가 직접 폴더를 추가하고 삭제할 수 있는 기능 복원
- 네비게이션 바에서 폴더 목록을 표시하고 이동, 삭제 가능
- "전체 파일", "기타 파일" 폴더는 삭제 불가능

3. system.js 코드 변경 사항
1) "기타 파일" 폴더 기본 제공 및 삭제 방지
- let folders = JSON.parse(localStorage.getItem("folders")) || { "전체 파일": [], "기타 파일": [] };

2)  loadFoldersFromStorage() 함수 수정
- 페이지 로드 시 "기타 파일" 폴더가 없을 경우 자동 생성

3) removeCategory() 함수 수정
- "기타 파일" 폴더 삭제 방지

4) classifyFile() 함수
- 파일 자동 분류 로직 알고리즘 함수
- 파일명이 특정 폴더와 매칭될 경우 해당 폴더와 "전체 파일"에 추가
- 매칭되지 않을 경우 "전체 파일"과 "기타 파일"에 추가

5) loadCategories() 함수 수정
- 폴더 목록 표시 함수
- "전체 파일"과 "기타 파일" 폴더에는 X 버튼 표시 안되도록 수정

4. 기대 효과
- 파일이 특정 폴더명과 일치하지 않아도 정리 가능
- 폴더 삭제 방지 기능을 통해 분류 체계 붕괴 방지
- 특정 키워드(폴더명)가 포함된 파일이 해당 폴더에 자동으로 추가
- 사용자가 폴더를 자유롭게 추가/삭제 가능
- 파일 검색, 다운로드, 삭제, 정렬 기능 유지하면서도 자동 분류 기능 추가
